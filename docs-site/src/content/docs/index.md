本文档站用于记录 Xuan ERP 项目微服务化重构的全部信息，包括项目背景、服务拆分、技术选型、权限体系、租户体系、CI/CD、部署运维、前端接入和功能组件设计。

## 当前定位

`D:\xuan-erp` 是未来微服务项目根目录，`docs-site` 是其中的 Astro Starlight 文档站子项目。

## 推荐阅读顺序

1. 项目概览：先了解系统目标和整体架构。
2. 架构设计：查看微服务拆分、技术栈、网关、服务通信、事件驱动。
3. 权限体系：理解 IAM、菜单权限、接口权限、列权限和前后端权限接入。
4. 租户体系：理解租户生命周期、tenantId 传递和各服务数据隔离。
5. 安全治理：理解服务间鉴权、内部接口保护和基础安全边界。
6. 后端开发：记录 DDD 分层、服务工程结构、服务文档模板、接口文档位置、数据库结构文档位置、接口契约、权限接入、数据库迁移。
7. 前端接入：记录路由、菜单、权限、页面开发规范。
8. 服务说明：逐个补充各微服务职责、数据归属、接口和事件。
9. 功能组件：沉淀类似 RuoYi 文档的功能说明。
10. 部署运维：记录 CI/CD、部署、监控、日志和常见问题。

## 文档维护原则

- 架构结论要写清楚背景、原因、取舍和落地方式。
- 服务文档要明确服务职责、数据所有权、权限清单、租户规则、事件发布与订阅。
- 功能组件文档要提供使用场景、前端调用、后端接口、权限要求、异常处理和注意事项。
- 涉及数据库结构、字段、索引、初始化数据或 Flyway migration 的内容，应记录版本顺序和迁移归属。
- 文档里如果出现规则冲突，迁移到新项目时必须先统一口径，不能原样照搬互相矛盾的旧描述。
- 重大架构决定使用 ADR 记录，例如 DDD、Flyway、权限模型、Nacos、服务通信、防 N+1。
