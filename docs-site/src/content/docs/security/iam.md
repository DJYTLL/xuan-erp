---
title: "IAM 权限中心"
---

IAM 是 Xuan ERP 的统一权限中心，集中管理用户、角色、权限、菜单、列权限、租户授权和当前用户授权快照。

## 核心原则

- 权限数据只归 IAM。
- 业务服务声明权限点，但不保存授权关系。
- Gateway 做入口认证。
- 业务服务执行本服务接口、资源和字段级校验。

## IAM 保存什么

| 数据 | 归属 | 说明 |
| --- | --- | --- |
| 用户 | IAM | 登录账号、密码凭证、用户状态 |
| 角色 | IAM | 租户内角色、系统角色、角色状态 |
| 菜单 | IAM | 全量菜单目录、菜单 code、路由关联 |
| 权限目录 | IAM | 全量 permission code、动作、所属服务 |
| 角色授权 | IAM | 角色拥有的菜单、接口权限、列权限 |
| 列权限 | IAM | pageKey、字段 code、可见性规则 |
| 用户租户关系 | IAM | 用户可进入哪些租户、在租户下有哪些角色 |
| 授权快照 | IAM | 登录后返回给前端和 Gateway 的权限摘要 |

业务服务不能保存“某个角色拥有哪些权限”这类授权关系。业务服务只维护自己的权限清单，例如 `xuan-sales` 维护 `sales:view`、`sales:create`、`sales:audit` 等定义，最终由受控流程同步进 IAM。

## 权限清单与授权关系

这两个概念必须分开：

| 概念 | 示例 | 谁维护 |
| --- | --- | --- |
| 权限清单 | 系统存在 `sales:audit` 这个权限点 | 业务服务声明，IAM 保存全量目录 |
| 授权关系 | 销售主管角色拥有 `sales:audit` | IAM 管理 |

所以“每个服务维护自己的权限清单”不是说每个服务保存角色授权，而是说每个服务对自己暴露的菜单、接口、字段负责，避免 IAM 团队手工猜每个业务服务有哪些权限。

## 权限同步时机

生产环境推荐同步发生在 CI/CD 或部署前置步骤中，而不是普通服务启动时直接写 IAM。

| 时机 | 是否推荐 | 原因 |
| --- | --- | --- |
| 代码提交后 CI 扫描 | 推荐 | 能在合并前发现权限码冲突、菜单缺失、pageKey 不一致 |
| 部署流水线同步 IAM | 推荐 | 有审计、有回滚、有环境隔离 |
| 服务启动时自检 | 推荐 | 只检查 IAM 中是否已有需要的权限，缺失则告警或阻止启动 |
| 服务启动时自动写 IAM | 不推荐 | 多实例并发、权限误删、生产变更不可审计，容易把启动行为变成数据变更 |

开发环境可以允许启动时自动注册权限，方便调试；测试和生产环境必须走受控同步。

## 权限码规范

权限码使用：

```text
领域:资源:动作
```

当资源和领域相同，可以简化为：

```text
领域:动作
```

示例：

| 权限码 | 说明 |
| --- | --- |
| `product:view` | 查看商品 |
| `product:create` | 新增商品 |
| `sales:audit` | 审核销售单 |
| `inventory:stock:adjust` | 调整库存 |
| `finance:receivable:view` | 查看应收明细 |

动作建议统一使用 `view`、`create`、`update`、`delete`、`audit`、`export`、`import`、`print`、`enable`、`disable`。

## 菜单、页面、权限关系

| 对象 | 示例 | 说明 |
| --- | --- | --- |
| 菜单 code | `trade:sales` | 菜单目录中的稳定编码 |
| 路由 name/path | `SalesOrderList` / `/sales/orders` | 前端路由入口 |
| pageKey | `sales-order` | 页面和列权限的稳定标识 |
| permission code | `sales:view` | 接口和按钮权限 |
| titleKey | `route.salesOrder` | 前端国际化标题 |

新增页面时，这些值必须一起设计，不能页面先上线、权限后补。

## super_admin 旁路

`super_admin` 可以绕过授权判断，但不能绕过基础安全校验：

- 必须校验 token 合法性。
- 必须校验租户状态。
- 必须保留审计日志。
- 接口仍然必须声明权限码。
- 列权限可以默认全量可见，但敏感导出仍建议保留审计。

## 服务拆分后的 IAM 边界

IAM 不应该承载所有业务判断。比如销售单能不能审核，是 `xuan-sales` 的业务规则；用户有没有 `sales:audit` 权限，是 IAM 授权结果加业务服务本地校验共同完成。

推荐链路：

```text
前端路由权限 -> Gateway 认证 -> 业务服务接口权限 -> 业务资源权限 -> 列权限过滤 -> 审计日志
```
